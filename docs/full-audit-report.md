## 🔴 КРИТИЧЕСКИЕ ПРОБЛЕМЫ БЕЗОПАСНОСТИ

### 1. Утечка реальных API ключей
**Файлы:** `.env`, `.env.example`  
**Риск:** Эксплуатация чужих API за ваш счёт, доступ к данным  
**Рекомендация:** 
- Немедленно отозвать/ротировать все ключи в `.env`
- Удалить `.env.example` из репозитория
- Добавить `.env` в `.gitignore`
- Использовать переменные окружения или secure vault

### 2. Отключенная аутентификация
**Файл:** `main.py:130`  
**Код:** `# app.add_middleware(AuthMiddleware)`  
**Риск:** Полный доступ к API без авторизации  
**Рекомендация:** Раскомментировать middleware, реализовать JWT или OAuth2

### 3. Хардкод учётных данных
**Файл:** `config.py`  
**Код:** `AUTH_USERNAME = "admin"`, `AUTH_PASSWORD = "admin2026"`  
**Риск:** Предсказуемые креды для brute-force  
**Рекомендация:** Вынести в `.env`, использовать хеширование паролей (bcrypt)

### 4. Mass Assignment уязвимость
**Файл:** `crud.py`  
**Код:** `setattr(agent, key, value)` для произвольных ключей  
**Риск:** Изменение `access_level`, `role`, `id` через API запрос  
**Рекомендация:** Белый список разрешённых полей для обновления

### 5. Path Traversal
**Файлы:** `api/agents_api.py`, `api/orchestrator_api.py`  
**Риск:** Чтение/запись произвольных файлов на сервере  
**Рекомендация:** Санитизация имён файлов, ограничение директории

### 6. XSS уязвимости
**Файл:** `static/app.js`  
**Код:** Множественные использования `innerHTML` без экранирования  
**Риск:** Выполнение произвольного JS в браузере пользователя  
**Рекомендация:** Использовать `textContent`, DOMPurify, или фреймворк с авто-экранированием

## 🟡 АРХИТЕКТУРНЫЕ ПРОБЛЕМЫ

### 1. Двойное управление состоянием
**Проблема:** Данные в SQLite + JSON файлах (`agents_state.json`, `project_memory.json`)  
**Риск:** Рассинхронизация, потеря данных, сложность отладки  
**Рекомендация:** Мигрировать всё в SQLite, использовать JSON только для кэша

### 2. Опасный async-паттерн
**Файл:** `api/episodes_api.py`  
**Код:** `asyncio.get_event_loop().run_until_complete()` внутри async функции  
**Риск:** Deadlock, блокировка event loop  
**Рекомендация:** Использовать `await` напрямую

### 3. Отсутствие транзакционной безопасности
**Файл:** `crud.py`  
**Проблема:** Нет `try/except` с `db.rollback()`  
**Риск:** Неполные транзакции, corrupted state  
**Рекомендация:** Обернуть все DB операции в транзакции

### 4. Монолитный оркестратор
**Файл:** `orchestrator/executor.py` (~1920 строк)  
**Проблема:** Дублирование CV-проверок, сложная логика  
**Риск:** Трудно поддерживать, тестировать, расширять  
**Рекомендация:** Разбить на модули: `pipeline_manager.py`, `cv_checker.py`, `critic_fixer.py`

### 5. Race Condition
**Файл:** `api/orchestrator_api.py`  
**Код:** `_running_pipelines = set()` без атомарности  
**Риск:** Параллельный запуск одинаковых пайплайнов  
**Рекомендация:** Использовать `asyncio.Lock()` или Redis

## 🟠 ПРОИЗВОДИТЕЛЬНОСТЬ

### 1. Незафиксированные версии зависимостей
**Файл:** `requirements.txt`  
**Проблема:** `>=` без верхних границ, отсутствуют `python-dotenv`, `pydantic`  
**Риск:** Breaking changes при деплое  
**Рекомендация:** Использовать `==` или `~=` для всех зависимостей

### 2. Частый polling фронтенда
**Файл:** `static/app.js`  
**Проблема:** `setInterval` каждые 5-15 секунд  
**Риск:** Нагрузка на сервер, батарея клиента  
**Рекомендация:** WebSocket или Server-Sent Events

### 3. Неатомарная запись JSON
**Файлы:** `api/tasks_api.py`, `api/discussion_api.py`, `api/project_api.py`  
**Проблема:** Прямая запись без временных файлов  
**Риск:** Потеря данных при краше  
**Рекомендация:** Использовать паттерн `tempfile.mkstemp()` + `os.replace()`
