# Animation Studio v2 — Идеи и Вопросы для Ментора

**Дата:** 04.04.2026
**Статус:** Реализованы Фазы 1-6 (Скелет, Агенты, Память, МЕД-ОТДЕЛ, HR, Инструменты).
**Проблема:** Текущая архитектура МЕД-ОТДЕЛА и промптов имеет фундаментальные недостатки, мешающие масштабированию.

---

## 🔴 Критические проблемы (Баги архитектуры)

### 1. Галлюцинации при эволюции промптов
**Симптом:** При эволюции Сценарист превратился в "Копирайтера". МЕД-ОТДЕЛ переписал системный промпт так, что изменилась роль агента, а не только инструкции.
**Причина:** Эволюция сейчас — это просто запрос к LLM: *"Исправь промпт"*. Модель может изменить всё что угодно.
**Риск:** Агенты теряют свою специализацию.

### 2. Отсутствие "Конституции Студии"
**Симптом:** Агенты забывают контекст проекта (например, что это YouTube-аниме по 3 минуты, а не арт-хаус), если это не повторять в каждом чате.
**Причина:** Глобальные правила хранятся только в голове у пользователя или размазаны по промптам агентов.
**Риск:** Рассинхрон между агентами. Критик судит по одним стандартам, Сценарист пишет по другим.

### 3. Непрозрачность МЕД-ОТДЕЛА
**Симптом:** Пользователь не видит, *почему* Критик отверг работу и *что именно* исправил МЕД-ОТДЕЛ.
**Причина:** Взаимодействие скрытое. Оценки теряются в логах или чате агента.
**Риск:** Невозможно отладить систему. Пользователь не доверяет "черному ящику".

---

## 💡 Предлагаемые решения (Roadmap v3)

### 1. Разделение: Конституция vs Роль
Ввести жесткую иерархию инструкций:
*   **Конституция (`constitution.md`):** Глобальные правила (Формат, Стиль, Тон). **Запрещено менять эволюцией.** Внедряется в каждый запрос автоматически.
*   **Роль (Role):** Личные навыки агента. **Эволюционирует.**
*   **Правила (Rules):** Динамические блоки, добавляемые МЕД-ОТДЕЛОМ при ошибках (например, `RULE: Не писать сцены длиннее 3 мин`).

### 2. Умная эволюция (Pattern Matching)
Вместо генерации промпта "с нуля", МЕД-ОТДЕЛ должен работать как инженер:
*   Создать **Библиотеку Паттернов** (`patterns.json`).
*   При ошибке "Слишком длинно" -> применять паттерн `EnforceBrevity`.
*   При ошибке "Забыл стиль" -> применять паттерн `AddStyleReminder`.
*   Это сделает эволюцию предсказуемой и контролируемой.

### 3. Групповой чат (Канал связи)
Сделать общение прозрачным:
*   Вкладка **"📢 Обсуждение"** в интерфейсе.
*   Там видны: Сообщение Агента -> Вердикт Критика -> Действие МЕД-ОТДЕЛА.
*   Пользователь может вмешаться: *"Критик прав, добавь экшена"*.

### 4. Иерархия агентов (Junior / Senior)
*   **Junior (Черновик):** Дешевая модель. Быстро генерирует идеи/структуру.
*   **Senior (Чистовик):** Дорогая модель. Проверяет черновик по Конституции, шлифует стиль.
*   *UI:* Переключатель `[Черновик] / [Финал]` или кнопка "Позвать ассистента".

## ❓ Вопросы к Ментору

1.  **Хранение Конституции:** Лучше хранить как отдельный файл, который подгружается в контекст, или как часть системного промпта каждого агента при инициализации?
2.  **Эволюция:** Стоит ли полностью запретить LLM генерировать текст промпта и перейти на систему "Конструктор правил" (добавление/удаление блоков)?
3.  **Групповой чат:** Технически лучше сделать это как отдельный канал (как в Slack/Discord) или как тред внутри задачи агента?
4.  **Память:** Как эффективно хранить "уроки" (Lessons), чтобы агент не тащил в контекст весь лог ошибок, а только релевантные? (RAG vs Суммаризация).
