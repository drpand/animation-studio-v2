# Rule Builder — Исправленный план

## 1. Схема данных `patterns.json`

Каждый паттерн — объект с 6 обязательными полями:

```json
{
  "patterns": [
    {
      "key": "duration_fix",
      "name": "Ограничение длительности",
      "rule_text": "[RULE] Длительность сцены строго до 3 минут.",
      "description": "Жёсткое ограничение для YouTube-формата.",
      "category": "format",
      "priority": 10
    }
  ]
}
```

| Поле | Тип | Описание |
|---|---|---|
| `key` | string | Уникальный идентификатор (snake_case, `[a-z_]+`) |
| `name` | string | Человекочитаемое название |
| `rule_text` | string | Текст правила, добавляемый в инструкции агента |
| `description` | string | Описание для UI |
| `category` | string | `format` / `style` / `narrative` / `consistency` |
| `priority` | int | 1–10, определяет порядок применения (10 = высший) |

## 2. Разрешение конфликтов

- **Последовательное применение**: правила добавляются в конец инструкций одно за другим.
- **Порядок**: при массовой выдаче правила сортируются по `priority` (по убыванию), затем по порядку добавления.
- **Дубликаты**: повторный `apply_pattern` того же `key` тому же агенту → отказ с `"error": "Правило уже применено"`.
- **Нет перезаписи role**: валидация запрещает паттернам, содержащим `role:`, применяться (защита от подмены роли агента).

## 3. API маршруты

Все эндпоинты с префиксом `/api/med-otdel/rules/`:

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/med-otdel/rules/patterns` | Список всех паттернов |
| `POST` | `/api/med-otdel/rules/apply` | Применить паттерн к агенту |
| `POST` | `/api/med-otdel/rules/remove` | Удалить паттерн у агента |
| `GET` | `/api/med-otdel/rules/:agent_id` | Список применённых правил агента |
| `POST` | `/api/med-otdel/rules/bulk-apply` | Массовое применение (сортировка по priority) |

## 4. Триггер применения

Два режима:

| Режим | Триггер | Механизм |
|---|---|---|
| **UI** | Пользователь выбирает паттерн в интерфейсе | `POST /api/med-otdel/rules/apply` |
| **Авто** | Critic обнаруживает повторяющуюся проблему → Fixer применяет паттерн | Critic возвращает `{"suggest_pattern": "key"}` → Fixer вызывает `apply_pattern()` |

## 5. Валидация

| Проверка | Что делает |
|---|---|
| **Дубликаты** | Отказ если `rule_text` уже в `instructions` агента |
| **Длина результата** | `len(new_instructions) <= 50000` символов — защита от раздувания |
| **Запрет `role`** | Если `rule_text` содержит `"role":` или `"role":` — отказ |
| **Ключ паттерна** | Regex `^[a-z][a-z0-9_]{1,49}$` — защита от injection |
| **Бесконечные циклы** | `applied_rules` хранит `pattern_key` — повторный apply того же key блокируется |
| **Макс правил на агента** | Не более 20 применённых правил одновременно |

## 6. Персистентность

`applied_rules` хранится в `agents_state.json` внутри каждого агента:

```json
{
  "writer": {
    "name": "Writer",
    "role": "...",
    "instructions": "...",
    "applied_rules": [
      {
        "pattern_key": "duration_fix",
        "rule_text": "...",
        "applied_at": "2026-04-04T12:00:00"
      }
    ]
  }
}
```

- Запись через atomic write (tempfile + `os.replace`) — уже реализовано в `_save_state()`.
- Thread-safe через `threading.Lock` — уже есть `_rule_lock`.

## 7. Rollback механизм

`remove_pattern(agent_id, pattern_key)`:

1. Находит `rule_text` по `pattern_key` в `patterns.json`
2. Удаляет `rule_text` из `instructions` агента
3. Удаляет запись из `applied_rules`
4. Чистит лишние пустые строки
5. Атомарно сохраняет состояние

Rollback = полное удаление следа правила. Никаких остатков.

## 10. Миграция данных

При старте приложения (`med_core.py` или `__init__.py`):

```python
def migrate_applied_rules():
    """Убедиться что у всех агентов есть applied_rules = []"""
    state = _load_state()
    changed = False
    for agent_id, agent in state.items():
        if "applied_rules" not in agent:
            agent["applied_rules"] = []
            changed = True
    if changed:
        _save_state(state)
```

- Вызывается **один раз** при инициализации.
- Все существующие агенты получают пустой `applied_rules`.
- Поле `rules` (старое) остаётся для обратной совместимости, но не используется.
