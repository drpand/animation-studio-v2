# Kie.ai Integration — Исправленный план

## Файлы
| Файл | Статус | Описание |
|---|---|---|
| `tools/kieai_tool.py` | **НОВЫЙ** | Клиент Kie.ai Z-Image (async) |
| `tools/comfyui_tool.py` | **ОСТАЁТСЯ** | Fallback если Kie.ai недоступен |
| `api/tools_api.py` | **ПРАВКА** | Сначала Kie.ai → при ошибке ComfyUI fallback |
| `config.py` | **ПРАВКА** | +KIEAI_API_KEY из `os.getenv` |
| `.env` | **ПРАВКА** | +KIEAI_API_KEY (ключ из .env, не хардкод) |
| `static/app.js` | **БЕЗ ИЗМЕНЕНИЙ** | Polling UI остаётся как было (спиннер) |

## Flow
```
POST createTask → async polling recordInfo (3 сек, 200 попыток) → success → download → save → return URL
```

## Fallback
```
Kie.ai fail → ComfyUI → fail → error
```

## Формат ответа
Оба возвращают `result_url` (URL изображения) — совместимы с `ToolResponse`.

## Тестирование
Добавлен эндпоинт `GET /api/tools/test-kieai` для проверки интеграции.

## Логирование
`print()` + логи в `kieai_tool.py` для отладки.

## Rate Limiting
Уже есть `RateLimiter` в `base_tool.py` — применяется ко всем инструментам.
